# 포트폴리오 랭크·대결 서비스 페이지 구성

## 1. 로그인 페이지

### 주요 기능

- GitHub 계정으로 로그인
- 일반 이메일 로그인
- 회원가입 페이지 이동
- 비밀번호 찾기

### 로그인 완료 후 처리

- 처음 가입한 사용자라면 프로필 설정 페이지로 이동
- 기존 사용자라면 메인 대시보드로 이동

### 입력 정보

- 사용자 이름
- 프로필 이미지
- 주 분야
- GitHub URL
- 자기소개

### 주 분야 예시

- 프론트엔드
- 백엔드
- 모바일
- 인공지능
- 데이터
- 게임 개발
- 클라우드·DevOps
- 보안
- 임베디드
- 디자인

### 주요 기능

- 사용자 이름 중복 확인
- GitHub URL 유효성 확인
- GitHub 계정 정보 연동
- 프로필 저장
- 관심 분야 선택

### 최초 랭크

신규 사용자는 플레이어 점수 `0점`, 플레이어 랭크 `F`부터 시작한다.

# 대결 시스템

## 11. 대결 로비 페이지

### 분야 선택

사용자는 대결할 분야를 선택한다.

- 자신의 포트폴리오가 등록된 분야만 선택 가능
- 선택한 분야에 맞는 포트폴리오를 대표 포트폴리오로 설정

### 대결 방식 선택

#### 랜덤 매칭

- 자신의 플레이어 랭크를 기준으로 상대 검색
- 최대 한 단계 차이의 랭크와 매칭
- 예: B 랭크 사용자는 A, B, C 랭크 사용자와 매칭 가능

#### 친구와 대결

- 사용자 이름으로 검색
- 친구 목록에서 선택
- 대결 초대 링크 생성
- 대결 초대 전송

### 대결 전 표시 정보

- 선택한 분야
- 사용할 포트폴리오
- 현재 플레이어 랭크
- 현재 플레이어 점수
- 승리 시 획득 점수
- 패배 시 차감 점수

## 12. 랜덤 매칭 페이지

### 주요 기능

- 매칭 시작
- 매칭 취소
- 검색 중 상태 표시
- 예상 매칭 범위 표시

### 매칭 조건

1. 같은 분야를 선택한 사용자
2. 대결 가능한 포트폴리오를 가진 사용자
3. 자신과 플레이어 랭크 차이가 한 단계 이하인 사용자
4. 현재 대결 중이 아닌 사용자
5. 자기 자신과는 매칭 불가

### 매칭 완료 후 표시

- 상대 사용자 이름
- 상대 플레이어 랭크
- 상대 승률
- 상대 포트폴리오 최고 랭크
- 선택 분야
- 대결 시작 버튼

분석 결과에 영향을 줄 수 있으므로 대결 시작 전에는 상대의 상세 포트폴리오 점수를 공개하지 않는다.

## 14. 대결 분석 진행 페이지

### 표시 정보

- 사용자 1 프로필
- 사용자 2 프로필
- 대결 분야
- 각 사용자의 포트폴리오
- 분석 진행률

### 분석 항목

각 포트폴리오를 항목별로 10점 만점으로 평가한다.

- 프로젝트 완성도
- 코드 구조
- 기술 활용
- 문서화
- 테스트
- 배포
- GitHub 활용

### 비교 점수 계산 방식

각 항목에서 높은 점수와 낮은 점수의 차이를 높은 점수를 받은 사용자에게 부여한다.

예시:

| 항목            | 사용자 1 | 사용자 2 | 획득 비교 점수 |
| --------------- | -------: | -------: | -------------- |
| 프로젝트 완성도 |        8 |        5 | 사용자 1 +3    |
| 코드 구조       |        4 |        5 | 사용자 2 +1    |
| 기술 활용       |        7 |        6 | 사용자 1 +1    |

이 경우 최종 비교 점수는 다음과 같다.

- 사용자 1: 4점
- 사용자 2: 1점
- 사용자 1 승리

두 사용자의 항목 점수가 같은 경우 해당 항목에서는 누구도 비교 점수를 얻지 않는다.

## 15. 대결 결과 페이지

### 최종 결과

- 승리
- 패배
- 무승부
- 사용자별 최종 비교 점수
- 플레이어 점수 변화

### 항목별 비교표

| 항목            | 사용자 1 점수 | 사용자 2 점수 | 차이 | 우세 사용자 |
| --------------- | ------------: | ------------: | ---: | ----------- |
| 프로젝트 완성도 |             8 |             5 |    3 | 사용자 1    |
| 코드 구조       |             4 |             5 |    1 | 사용자 2    |
| 기술 활용       |             7 |             6 |    1 | 사용자 1    |
| 문서화          |             5 |             5 |    0 | 동점        |
| 테스트          |             4 |             7 |    3 | 사용자 2    |
| 배포            |             8 |             6 |    2 | 사용자 1    |
| GitHub 활용     |             9 |             7 |    2 | 사용자 1    |

### 승패 결정

모든 항목에서 획득한 비교 점수를 합산한다.

- 최종 비교 점수가 높은 사용자가 승리
- 두 사용자의 최종 비교 점수가 같으면 무승부

### 플레이어 점수 변화

| 결과   | 점수 변화 |
| ------ | --------: |
| 승리   |        +8 |
| 패배   |        -3 |
| 무승부 |         0 |

플레이어 점수는 최소 0점, 최대 100점을 넘지 않는다.

### 사용자별 피드백

#### 좋았던 점

상대보다 높은 평가를 받은 항목과 그 이유를 보여준다.

#### 개선할 점

상대보다 낮은 평가를 받은 항목과 개선 방법을 보여준다.

# 핵심 사용자 흐름

## 포트폴리오 분석 흐름

`메인 대시보드 → 포트폴리오 등록 → 분야 선택 → GitHub 링크 입력 → 저장소 확인 → 분석 진행 → 결과 확인 → 플레이어 점수 반영`

## 랜덤 대결 흐름

`대결 로비 → 분야 선택 → 대표 포트폴리오 선택 → 랜덤 매칭 → 상대 매칭 → 비교 분석 → 승패 결정 → 플레이어 점수 반영`

## 친구 대결 흐름

`대결 로비 → 친구와 대결 → 사용자 선택 → 분야 및 포트폴리오 선택 → 초대 전송 → 상대 수락 → 비교 분석 → 결과 확인`

# 구현 시 필요한 핵심 정책

## 플레이어 점수 정책

- 신규 사용자는 0점으로 시작한다.
- 플레이어 점수는 0점 미만으로 내려가지 않는다.
- 플레이어 점수는 100점을 초과하지 않는다.
- 포트폴리오 분석 점수와 대결 결과로 플레이어 점수가 변경된다.
- 같은 포트폴리오를 반복 분석해 점수를 무한히 얻지 못하도록 제한이 필요하다.

권장 방식은 동일 커밋을 재분석한 경우 플레이어 점수를 지급하지 않고, 이전 분석보다 높은 랭크를 달성한 경우 차액만 지급하는 것이다.

## 대결 정책

- 동일한 포트폴리오끼리는 대결할 수 없다.
- 자기 자신과는 대결할 수 없다.
- 랜덤 매칭은 최대 한 랭크 차이까지만 허용한다.
- 이미 대결한 두 포트폴리오의 코드가 변경되지 않았다면 반복 점수 획득을 제한한다.
- 비정상적인 반복 대결은 랭크 점수에 반영하지 않는다.
- 분석이 완료된 포트폴리오만 대결에 사용할 수 있다.

## 무승부 정책

최종 비교 점수가 같은 경우 무승부로 처리한다.

- 승리 횟수 증가 없음
- 패배 횟수 증가 없음
- 플레이어 점수 변화 없음
- 승률 계산 시 무승부를 전체 대결 횟수에 포함할지 정책을 별도로 정해야 한다.

권장 승률 계산식은 다음과 같다.

`승률 = 승리 횟수 ÷ (승리 횟수 + 패배 횟수) × 100`

무승부는 승률 계산에서 제외한다.
